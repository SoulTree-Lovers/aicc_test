# RAG 콜센터 UX 설계 문서

## 1) 문서 목적
본 문서는 RAG(Retrieval-Augmented Generation) 기반 콜센터 어시스턴트의 핵심 사용자 여정과 대화 UX 원칙을 정의한다. 목표는 **빠른 초기 응답**, **정확한 문제 분류**, **최소 재질문으로 해결**, **필요 시 신속한 상담원 이관**이다.

---

## 2) 핵심 사용자 여정 정의

### 여정 A. 요금/청구 문의
- 대표 질문: “이번 달 요금이 왜 늘었나요?”, “청구서 내역을 설명해 주세요.”
- 핵심 과업: 청구 내역 확인, 증감 사유 설명, 납부/미납 상태 안내, 이의제기 절차 안내.

### 여정 B. 서비스 장애/복구 문의
- 대표 질문: “인터넷이 안 돼요.”, “장애 복구 언제 되나요?”
- 핵심 과업: 장애 여부 판별(개별/지역), 기본 점검 가이드, 복구 ETA 안내, 긴급 이관.

### 여정 C. 계약/해지/변경 문의
- 대표 질문: “약정이 언제 끝나나요?”, “해지하려면 어떻게 하나요?”, “요금제를 바꾸고 싶어요.”
- 핵심 과업: 계약 상태 조회, 변경/해지 조건 안내, 위약금/혜택 영향 설명, 실행 경로 안내.

---

## 3) 여정별 UX 명세 표

### 3-1. 요금/청구 문의

| 항목 | 내용 |
|---|---|
| 사용자 의도 | 청구 금액/증감 원인 파악, 납부 상태 확인, 이의 신청 방법 확인 |
| 필요한 슬롯 | 고객번호, 본인확인 정보(생년월일/인증수단), 청구월(YYYY-MM), 서비스 종류(모바일/인터넷/IPTV), 문의 유형(증액/이중청구/할인누락) |
| 금지 질문 | 카드 전체번호, 계좌 비밀번호, 주민등록번호 전체, 불필요한 반복 신원질문 |
| 기대 응답시간 | 초기 응답 3초 이내, 상세 근거 응답 8초 이내 |
| 성공 기준 (CSAT/해결률) | CSAT 4.5/5 이상, 셀프 해결률 75% 이상, 동일 이슈 재문의율 15% 이하 |

### 3-2. 서비스 장애/복구 문의

| 항목 | 내용 |
|---|---|
| 사용자 의도 | 현재 장애 원인/범위 확인, 복구 예상시간 확인, 즉시 가능한 조치 수행 |
| 필요한 슬롯 | 고객번호 또는 회선번호, 서비스 종류, 장애 시작 시각, 증상 유형(무응답/속도저하/간헐끊김), 지역/주소(시·구 단위), 장비 상태(모뎀/공유기 LED) |
| 금지 질문 | 내부 시스템 코드 노출 요구, 기술적으로 무관한 개인정보, 과도한 로그 수집(사용자 비전문 영역) |
| 기대 응답시간 | 초기 응답 2초 이내, 장애 판별 결과 5초 이내, 복구 ETA 안내 10초 이내 |
| 성공 기준 (CSAT/해결률) | CSAT 4.4/5 이상, 1턴 원인 분류 정확도 90% 이상, 원격 셀프복구 성공률 60% 이상 |

### 3-3. 계약/해지/변경 문의

| 항목 | 내용 |
|---|---|
| 사용자 의도 | 약정/위약금/변경 가능 여부 확인, 해지/변경 절차 이해 및 실행 |
| 필요한 슬롯 | 고객번호, 계약 서비스 종류, 계약 상태(약정/무약정), 희망 처리일, 변경 유형(업/다운그레이드/부가서비스), 본인확인 수단 |
| 금지 질문 | 해지 방어를 위한 압박성 질문, 무관한 가족관계/직장정보, 추가 영업성 질문(사용자 거부 후) |
| 기대 응답시간 | 초기 응답 3초 이내, 조건 요약 7초 이내, 실행 경로 제시 10초 이내 |
| 성공 기준 (CSAT/해결률) | CSAT 4.3/5 이상, 의도 대비 정확 안내율 92% 이상, 처리 경로 이탈률 10% 이하 |

## 4) 대화 시작/재질문 템플릿

### 4-1. 초기 안내 문구
> 안녕하세요, 고객님의 문의를 빠르게 해결해드리겠습니다.  
> **요금/청구, 서비스 장애, 계약/변경/해지** 중 어떤 도움이 필요하신가요?  
> 고객번호를 알고 계시면 함께 알려주시면 더 정확하게 안내해드릴 수 있습니다.

### 4-2. 재질문(clarification) 템플릿

#### 공통 템플릿
- “정확한 확인을 위해 **[누락 슬롯]**이 필요합니다. 확인 가능하실까요?”
- “말씀하신 내용이 **[해석 A]**인지, 아니면 **[해석 B]**인지 확인 부탁드립니다.”
- “지금 바로 해결 가능한 경로로 안내드리기 위해 **[필수 정보 1~2개]**만 추가로 알려주세요.”

#### 여정별 템플릿 예시
- 요금/청구: “어느 달 청구서 기준인지 확인을 위해 **청구월(예: 2026-03)**을 알려주세요.”
- 장애/복구: “장애 범위를 확인하려면 **서비스 종류(인터넷/모바일 등)**와 **지역(시·구)** 정보가 필요합니다.”
- 계약/변경/해지: “정확한 조건 안내를 위해 **현재 약정 여부**와 **희망 처리일**을 알려주세요.”

## 5) 답변 톤 가이드 및 불가 응답 대안

### 5-1. 톤 가이드
- **공손**: 존댓말 유지, 책임 회피 표현 금지 (“저희가 확인해보겠습니다”).
- **간결**: 1응답 3문장 내 핵심 요약 + 필요 시 단계형 안내.
- **행동지향**: 매 응답 끝에 다음 행동 제시 (“지금 고객번호를 입력해 주세요”, “복구 알림 신청을 도와드릴까요?”).

### 5-2. 불가 응답 시 대안(상담원 연결 안내)
다음 조건 중 하나라도 해당하면 즉시 상담원 연결 옵션을 제시한다.
- 본인확인 실패 또는 보안상 민감 처리 필요
- 반복 2회 이상 재질문에도 핵심 슬롯 미확보
- 시스템 조회 실패/장애로 확정 답변 불가
- 감정 고조(강한 불만/분쟁) 또는 법적 민감 이슈

#### 연결 안내 템플릿
> 정확하고 안전한 처리를 위해 상담원 연결이 필요합니다.  
> 지금 바로 연결해드릴까요? 원하시면 **콜백 예약(희망 시간 선택)**도 가능합니다.

## 6) UX 검증 기준 (KPI)

| 지표 | 정의 | 목표 | 측정 방법 |
|---|---|---|---|
| 첫 응답 시간 (FRT) | 사용자 첫 발화 후 어시스턴트 첫 응답까지 시간 | 3초 이하 (P95) | 대화 로그 타임스탬프 차이 |
| 1턴 해결률 | 첫 질의-첫 답변 1턴 내 해결 완료 비율 | 55% 이상 | 해결 태그 + 후속 발화 유무 |
| 재질문 비율 | 총 세션 중 clarification 질문 발생 비율 | 35% 이하 | 재질문 템플릿 트리거 로그 |
| 상담원 이관률 | 전체 세션 중 상담원 이관 비율 | 20% 이하 | 이관 이벤트 로그 |

### 운영 해석 가이드
- FRT가 목표 초과 시: 검색 인덱스/캐시/프롬프트 길이 최적화 우선.
- 1턴 해결률 저하 시: 슬롯 설계 단순화 및 FAQ 매핑 재학습.
- 재질문 비율 과다 시: 필수 슬롯 재정의(“있으면 좋은 정보”와 분리).
- 이관률 급증 시: 장애/청구 피크 시즌 시나리오 별도 라우팅 추가.

---

## 7) 적용 우선순위
1. 초기 안내 문구 + 여정 분류 정확도 개선
2. 슬롯 최소셋 정비 및 재질문 템플릿 적용
3. KPI 대시보드 연동(FRT/1턴 해결률/재질문/이관률)
4. 월간 CSAT·해결률 기반 문구/정책 튜닝
